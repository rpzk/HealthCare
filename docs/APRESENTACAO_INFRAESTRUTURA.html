Apresentação de Infraestrutura e Mitigação de Riscos - Propay

O Desafio de Compliance & A Solução

Cenário Atual (Formulários Genéricos)

  • • Risco Crítico LGPD: Dados sensíveis de saúde trafegando sem criptografia ponta a ponta adequada para o setor.
  • • Ausência de Rastreabilidade: Impossibilidade de auditar quem acessou ou alterou um dado (Falta de Logs).
  • • Insegurança Jurídica: Sem integração com certificação digital ICP-Brasil.

Cenário Proposto (HealthCare)

  • • Privacy by Design: Controle de Acesso Baseado em Função (RBAC) – O RH não vê o que o médico vê.
  • • Auditoria Completa: Logs imutáveis de cada acesso, edição ou exclusão.
  • • Segurança CFM: Assinatura Digital A1 nativa no prontuário.

Arquitetura Cloud-Native

Aplicação

Next.js 14 com React 18, arquitetura App Router moderna e foco em desempenho e experiência do profissional de saúde.

Dados

PostgreSQL 15 com Prisma ORM, criptografia em repouso e segregação por tenant, preparada para Azure Database for PostgreSQL.

Integrações

Redis para cache e filas, Gotenberg para PDFs assinados e conectores FHIR/LAB para interoperabilidade clínica.

Toda a infraestrutura é empacotada em Docker, permitindo deploy consistente em Azure ou outros provedores cloud.

Custos e Mitigação de Riscos

Risco de não agir

  • • Multas LGPD, ações trabalhistas e reputação afetada.
  • • Ausência de trilhas de auditoria para incidentes de segurança.

Proposta HealthCare

  • • Modelo de licenciamento previsível, sem surpresas de infraestrutura.
  • • Roadmap alinhado às necessidades da Propay, com suporte técnico próximo.

Resumo para Decisão

A proposta do HealthCare é reduzir risco regulatório, aumentar a governança clínica e preparar a Propay para um crescimento seguro em saúde digital, com uma arquitetura que fala a língua de tecnologia e de compliance ao mesmo tempo.

Próximos passos: validar aderência de fluxos internos, homologar ambiente e planejar rollout gradual por unidades.

🎯 Recomendações Estratégicas

Quando escolher Local (On-Premise)

✅ Recomendado para:

  • ✓ Clínicas/hospitais com volume médio a alto (> 100 pacientes/dia)
  • ✓ Orçamento para investimento inicial disponível
  • ✓ Equipe técnica local ou contrato de suporte
  • ✓ Preocupação com custos recorrentes de longo prazo
  • ✓ Necessidade de controle total dos dados
  • ✓ Internet instável ou cara

Quando escolher Azure Cloud

☁️ Recomendado para:

  • ✓ Clínicas pequenas iniciando operação (< 50 pacientes/dia)
  • ✓ Sem orçamento para investimento inicial
  • ✓ Sem equipe técnica local
  • ✓ Necessidade de alta disponibilidade (99.9%+)
  • ✓ Crescimento rápido esperado
  • ✓ Múltiplas unidades/filiais
  • ✓ Compliance rigoroso (certificações Azure)

Solução Híbrida

💡 Recomendação: Para clínicas médias, considere uma solução híbrida:
• Servidor Local: Aplicação + banco de dados
• Azure Backup: Backup offsite automatizado
• Azure CDN: Entrega rápida de assets
Custo estimado: R$ 15-20k (1º ano) | R$ 8-12k (anos seguintes)

Para clínicas médias e grandes, a implementação local (on-premise) oferece o melhor custo-benefício, com ROI em 2-4 meses e economia de 47% a 85% em relação ao Azure.
